## 絵文字による発話者判別ルール（重要・再検証済み）

hrk/トーク/[LINE] 百目とのトーク.txt では夫婦間の**ロールプレイ絵文字**が多用されている。

- **「（＾◇＾）」**: **はるか（百目）を表す**キャラクター絵文字
- **「（＾ν＾）」「（>＾ν＾<）」**: **ユーザー（ひでお／犬コロ）を表す**キャラクター絵文字
- **「🐶」「🐕」**: ユーザーの補助マーカー（犬コロ）

**ロールプレイ用法**: どちらの話者もこれらの絵文字を借りて **相手役／自分役のセリフを交互に演じる** ケースが多数（例: 36293・36670・37326・45742-45751 等で確認）。

**判別の原則**:
- **話者ラベル（LINE上の名前）を最優先**する（絵文字だけでラベル逆転を決めつけない）
- 絵文字は**そのセリフが誰のキャラを演じているか**の指標に留まる
- 「ひでお」ラベル+「（＾◇＾）〜」の発言 = **ユーザー本人がはるか役を演じたセリフ**（例: 「別居したい」「恋したい」等）→ **リスク側で扱う**（ユーザーの実発言）
- 「百目」ラベル+「（＾ν＾）〜」の発言 = **はるか本人がユーザー役を演じたセリフ**（例: 「（＾ν＾）まえかの」等）→ 補強材料

**戦略的含意**: 45814-45817「（＾◇＾）別居したい」等はユーザー自身の深夜発言（はるか役ロールプレイの体裁だが、発話行為はユーザー）。従って`08-user-risks.md` R1のリスク扱いが正当。ただし絵文字＝配偶者役の演出であり、真意ではない酔言／戯言の解釈余地は大きい
